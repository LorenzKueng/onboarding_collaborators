# New Project Scaffold

Create a new research project folder with the standard structure used in Lorenz Kueng's empirical research projects.

Check whether the `--secure` flag was provided. If yes, use Case 2 (sensitive data on secure server) throughout. If no flag, use Case 1 (all public data).

## What Gets Created

### Case 1: All data is public (default)

```
[project-name]/
├── .gitignore
├── CLAUDE.md                   ← project instructions + first-time setup (symlinks, paths)
├── MEMORY.md
├── README.md                   ← GitHub-facing overview (displayed on repo landing page)
├── REPLICATION.md              ← replication instructions for journal data editor only
├── code/
│   ├── stata/
│   │   ├── master.do           ← run this to execute the full pipeline
│   │   └── 00_setup.do         ← all globals; sourced at top of every do-file
│   ├── R/
│   └── python/
├── data/
│   ├── raw/
│   │   └── _source.md
│   └── _derived/
├── output/
│   ├── figures/
│   ├── tables/
│   └── other_output/
├── tex/                        ← symlink → [project-name]_Overleaf/ (set up manually)
├── documents/
├── correspondence/
│   └── referee2/
├── progress_logs/
│   └── YYYY-MM-DD_description.md
└── _scratch/

[project-name]_Overleaf/        ← separate folder (e.g. Dropbox/Apps/ShareLaTeX/)
├── [project-name].tex
├── [project-name]_slides.tex
├── sections/
│   ├── 00_abstract.tex
│   └── 01_introduction.tex
└── output/                     ← symlink → [project-name]/output/
```

### Case 2: Sensitive data on secure server (--secure)

Same as Case 1, plus `_scratch/output/` for local dev runs (so live code never writes to the protected `output/` folder). The server directory is documented in `CLAUDE.md` but must be set up manually on the server.

```
[project-name]/
...same as Case 1, except _scratch/ also contains:
└── _scratch/
    └── output/                 ← $output_dev — local dev runs write here

[project-name]_server/          ← set up manually on secure server
├── code/                       ← copy of [project-name]/code/ via rsync
├── data/
│   ├── secure/                 ← $data_secure — never leaves server
│   │   └── _source.md
│   ├── public/                 ← copy of [project-name]/data/raw/ via rsync
│   └── _derived/               ← all intermediate files (any that touched secure data)
├── output/
│   ├── secure/                 ← $output_secure — awaiting third-party vetting
│   └── vetted/                 ← approved output; rsync'd to [project-name]/output/
├── logs/
└── _scratch/
```

## Execution

### Step 1: Create folder structure

**Run for both cases:**

```bash
mkdir -p [project-name]/code/stata
mkdir -p [project-name]/code/R
mkdir -p [project-name]/code/python
mkdir -p [project-name]/data/raw
mkdir -p [project-name]/data/_derived
mkdir -p [project-name]/output/figures
mkdir -p [project-name]/output/tables
mkdir -p [project-name]/output/other_output
mkdir -p [project-name]/documents
mkdir -p [project-name]/correspondence/referee2
mkdir -p [project-name]/progress_logs
mkdir -p [project-name]/_scratch
```

**If `--secure` flag provided, also run:**

```bash
mkdir -p [project-name]/_scratch/output
```

---

### Step 2: Create `.gitignore` and `.dropboxignore`

#### `.gitignore`

Same for both cases — `tex/` is always excluded since it is a symlink to the Overleaf folder (contents tracked by Overleaf, not Git).

```
# Data files — synced by cloud storage, not Git
*.dta
*.csv
*.xlsx
*.xls
data/raw/
data/_derived/

# Output files — regenerated by code, not tracked
output/

# Scratch files
_scratch/

# Overleaf symlink — contents tracked by Overleaf, not Git
tex/

# OS files
.DS_Store
Thumbs.db
```

#### `.dropboxignore`

Same for both cases. Prevents per-machine symlinks and temporary files from syncing to other co-authors:

```
# Per-machine symlinks — each co-author creates locally; do not sync
tex

# Temporary files — not for sharing
_scratch
```

### Step 3: Create `CLAUDE.md`

**If no `--secure` flag (Case 1):**

```markdown
# Project: [Project Name]

## What This Project Is
[One paragraph description of the research question]

## Key Files
- `code/stata/master.do` — run this to execute the full pipeline
- `code/stata/00_setup.do` — sets all path globals; sourced at top of every do-file

## Stata Globals
- `$raw` — raw data folder (`data/raw/`)
- `$derived` — intermediate processed datasets (`data/_derived/`)
- `$output` — figures, tables, and other output (`output/`)
- `$OverleafRoot` — local path to Overleaf folder (set per user in `00_setup.do`; use this to read `.tex` files)

## Conventions
- Raw data is never modified — all processing in `_derived/`
- Every do-file starts by running `00_setup.do`
- Cross-language replication: Stata results replicated in R and Python

## Co-authors
[Names and roles if applicable]

---

## Session Start
At the start of each session, read `code/stata/00_setup.do` to know the current path globals (`$ProjectRoot`, `$OverleafRoot`, etc.).

---

## First-Time Setup (each co-author, once per machine)

### 1. Set your paths in `code/stata/00_setup.do`
Add a block for your username with your `$ProjectX` path and `$OverleafRoot` path.
Claude uses `$OverleafRoot` to read `.tex` files directly — no symlink needed for that purpose.

### 2. Create symlinks (per machine — excluded from Dropbox via `.dropboxignore`)

`[project-name]_Overleaf/output/` must point to `[project-name]/output/` so figures and
tables update automatically when you recompile in Overleaf. `[project-name]/tex/` gives
Claude a short path to `.tex` files and is excluded from Dropbox sync via `.dropboxignore`.

**Windows** (run Command Prompt as Administrator):
```
mklink /D "[path to ProjectX_Overleaf]\output" "[path to ProjectX]\output"
mklink /D "[path to ProjectX]\tex" "[path to ProjectX_Overleaf]"
```

**Mac** (run in Terminal):
```
ln -s "[path to ProjectX]/output" "[path to ProjectX_Overleaf]/output"
ln -s "[path to ProjectX_Overleaf]" "[path to ProjectX]/tex"
```
```

---

**If `--secure` flag provided (Case 2):**

```markdown
# Project: [Project Name]

## What This Project Is
[One paragraph description of the research question]

## Data
- **Public data** — `data/raw/` (local, cloud-synced, not Git)
- **Sensitive data** — `[project-name]_server/data/secure/` (secure server only, never synced)
- Any derived data that touches secure data lives on the server only

## Key Files
- `code/stata/master.do` — run this to execute the full pipeline
- `code/stata/00_setup.do` — sets all path globals; sourced at top of every do-file

## Stata Globals
- `$data_public` — public raw data (`data/raw/`)
- `$derived_public` — intermediate files from public data only (`data/_derived/`)
- `$data_secure` — sensitive data on server (`[project-name]_server/data/secure/`)
- `$derived` — intermediate files on server (any that touched secure data)
- `$output` — protected output folder, Overleaf-facing (`output/`)
- `$output_dev` — dev run output, never touches protected folder (`_scratch/output/`)
- `$output_secure` — analysis output on server, awaiting third-party vetting
- `$OverleafRoot` — local path to Overleaf folder (set per user in `00_setup.do`; use this to read `.tex` files)

## Conventions
- Raw data is never modified — all processing in `_derived/`
- Local code runs always write to `$output_dev`, not `$output`
- `$output` is populated only by manual rsync from server after third-party approval
- Every do-file starts by running `00_setup.do`
- Cross-language replication: Stata results replicated in R and Python

## Co-authors
- Full access (public + secure data): [names]
- Public data only: [names]

## First-Time Setup (each co-author, once per machine)

### 1. Set your paths in `code/stata/00_setup.do`
Add a block for your username with your `$ProjectX` path and `$OverleafRoot` path.
Full-access co-authors also set `$ProjectX_server`.

### 2. Create symlinks (per machine — excluded from Dropbox via `.dropboxignore`)

`[project-name]_Overleaf/output/` must point to `[project-name]/output/` so figures and
tables update automatically when you recompile in Overleaf. `[project-name]/tex/` gives
Claude a short path to `.tex` files and is excluded from Dropbox sync via `.dropboxignore`.

**Windows** (run Command Prompt as Administrator):
```
mklink /D "[path to ProjectX_Overleaf]\output" "[path to ProjectX]\output"
mklink /D "[path to ProjectX]\tex" "[path to ProjectX_Overleaf]"
```

**Mac** (run in Terminal):
```
ln -s "[path to ProjectX]/output" "[path to ProjectX_Overleaf]/output"
ln -s "[path to ProjectX_Overleaf]" "[path to ProjectX]/tex"
```

### 3. Server setup (full-access co-authors only)
Create the directory structure on the secure server:
```bash
mkdir -p [project-name]_server/code
mkdir -p [project-name]_server/data/secure
mkdir -p [project-name]_server/data/public
mkdir -p [project-name]_server/data/_derived
mkdir -p [project-name]_server/output/secure
mkdir -p [project-name]_server/output/vetted
mkdir -p [project-name]_server/logs
mkdir -p [project-name]_server/_scratch
```

### 4. Syncing code and data to server (SSH case)
Push code and public data before any server job:
```bash
rsync -av "[path to ProjectX]/code/"     user@server:[project-name]_server/code/
rsync -av "[path to ProjectX]/data/raw/" user@server:[project-name]_server/data/public/
```
Pull vetted output after third-party approval:
```bash
rsync -av user@server:[project-name]_server/output/vetted/ "[path to ProjectX]/output/"
```
```

### Step 6: Create `code/stata/00_setup.do`

**If no `--secure` flag (Case 1):**

```stata
********************************************************************************
*** SETUP — [Project Name]
*** Run this at the top of every do-file
********************************************************************************

*** === Each user sets these ===
if "`c(username)'" == "[your username]" {
    global ProjectX     "[path to project folder]"
    global OverleafRoot "[path to Dropbox/Apps/ShareLaTeX/ProjectX_Overleaf]"
}

*** === Derived globals (same for all users) ===
global raw        "$ProjectX/data/raw/"
global derived    "$ProjectX/data/_derived/"
global output     "$ProjectX/output/"
```

**If `--secure` flag provided (Case 2):**

```stata
********************************************************************************
*** SETUP — [Project Name]
*** Run this at the top of every do-file
********************************************************************************

*** === Each user sets these ===
if "`c(username)'" == "[your username]" {
    global ProjectX        "[path to project folder]"
    global OverleafRoot    "[path to Dropbox/Apps/ShareLaTeX/ProjectX_Overleaf]"
    global ProjectX_server "[path to secure server folder]"  // omit if public-only access
}

*** === Public data (local) ===
global data_public    "$ProjectX/data/raw/"
global derived_public "$ProjectX/data/_derived/"

*** === Secure data (server — omit if no server access) ===
global data_secure    "$ProjectX_server/data/secure/"
global derived        "$ProjectX_server/data/_derived/"

*** === Output ===
global output         "$ProjectX/output/"           // protected, Overleaf-facing
global output_dev     "$ProjectX/_scratch/output/"  // dev runs only — never touch $output
global output_secure  "$ProjectX_server/output/secure/"
```
